## Сессия 1 (2026-03-17)

### L1: Python недоступен — использовать Node.js
**Проблема**: Plan предполагал `python-docx`, но Python — только Windows Store stub.
**Решение**: Node.js + npm пакет `docx`. Работает отлично.
**Правило**: На этой машине все скрипты — через Node.js.

### L2: MCP playwright — правильный синтаксис cmd на Windows
**Проблема**: `args: ["C/", "npx", ...]` не работает.
**Правило**: Первый аргумент cmd.exe всегда `/c` (строчная, со слешем вперёд).

### L3: Word-файл нельзя перезаписать пока открыт
**Проблема**: `EBUSY` при попытке перезаписать .docx пока он открыт в Word.
**Решение**: Генерировать в новый файл (`_v2.docx`), затем переименовывать.

### L4: Для геймдизайна — начинать с рынка, не с идеи
**Открытие из исследования**: Большинство игр не находят издателя из-за игнорирования рынка.
**Правило**: Фаза 0 (рынок + издатели) — обязательна и идёт первой.

### L5: Данные по цифровым портам — осторожно с пандемийными пиками
**Открытие**: Пики 2020 (Ticket to Ride: 2115 игроков) — аномалия. В 2026 — 180.
**Правило**: При оценке рынка цифровых портов использовать данные 2024-2026, не пики 2020.
