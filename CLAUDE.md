## Как работать в этом проекте

### Доступные права (расширенные)
- WebSearch — использовать для BGG, Steam, Cardboard Edison, Kicktraq
- Чтение/запись файлов — все файлы проекта
- Bash/Node.js — обработка данных, скрипты
- Supabase MCP — если понадобится база данных

### Правила работы
1. **Всегда начинать с Фазы 0** — рынок, не механики
2. **XML-теги** для структурированных промптов: `<rules>`, `<cards>`, `<constraints>`
3. **Субагенты** для исследований — не засорять главный контекст
4. **Сохранять артефакты** в `docs/` или `data/` сразу после генерации
5. **Обновлять этот файл** при смене фокуса или завершении фазы

### Структура папок (создать при старте)
```
docs/
  game_design_doc.md     ← главный дизайн-документ
  rulebook_draft.md      ← черновик правил
  competitors.md         ← анализ конкурентов
  publisher_targets.md   ← список издателей
data/
  cards_database.csv     ← база карточек
  playtest_log.md        ← журнал плейтестов
pitch/
  pitch_deck.md          ← питч-материалы
tasks/
  todo.md                ← текущий план работ
```
