## Как работать в этом проекте

### Доступные права (расширенные)
- WebSearch — использовать для BGG, Steam, Cardboard Edison, Kicktraq
- Чтение/запись файлов — все файлы проекта
- Bash/Node.js — обработка данных, скрипты
- Supabase MCP — если понадобится база данных

### Правила работы
1. **Всегда начинать с Фазы 0** — рынок, не механики
2. **XML-теги** для структурированных промптов: `<rules>`, `<cards>`, `<constraints>`
3. **Субагенты** для исследований — не засорять главный контекст
4. **Сохранять артефакты** в `docs/` или `data/` сразу после генерации
5. **Обновлять этот файл** при смене фокуса или завершении фазы

### Граничные условия UI прототипа (ОБЯЗАТЕЛЬНО)
- **Слоты инструментов одинаковой высоты**: все `.card-slot` в одной grid-строке должны иметь одинаковую высоту. Использовать `align-items: stretch` (default) на `#tool-section`. Подсказки и кнопки — `align-self: center`, но НЕ весь контейнер.
- **Не расширять `#center-col`**: любой элемент внутри `#center-col` не должен быть шире игровой сетки (~460–487px). Ширину задаёт `#grid-wrap`. Никаких отдельных grid/flex с `1fr` колонками внутри `#center-col` если в них есть широкие элементы (кнопки с длинным текстом). Проверять через `#tool-section.offsetWidth === #grid-wrap.offsetWidth`.
- **Правая панель (`#info-col`) — не сжимать ниже 50%**: `#info-col` имеет `flex: 1`. Не добавлять на `#game-area` или его детей стили, которые уменьшают доступное пространство для `#info-col` менее чем на 50% ширины `#app`.
